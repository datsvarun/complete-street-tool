The complete street tool (CST) takes on the task of creating urban design plans for streets. Primarily focused on the Indian Urban streets and built upon the foundation of Indian road standards developed by the Indian Road Congress (IRC).
The application/plugin has a few main objectives
- Creating code formatted standardized street sections from IRC Codes.
- Automatic, dynamic and rule-based generation of these street sections into a plan fit for urban design and urban planning.
- Enabling export of these plans to professional software for detailing and construction.

The tools stems from the poor pedestrian and [divyang](https://www.pib.gov.in/PressReleasePage.aspx?PRID=2169293&reg=3&lang=2) accessibility of Indian streets, with an easy to use interface for budding engineers, architects and urban practitioners to generate code conforming street designs for implementation throughout the cities. 

It builds off the work done in [3D Street](https://3dstreet.app/) and [StreetMix](https://streetmix.net/). While being exciting tools, they lack the representation of localisation of these streets into actual neighbourhoods in a plan format. 3D Street has some capability but is paywalled and insufficient with regards to intersections/junctions.

Professional tools like[ESRI CityEngine](https://www.esri.com/en-us/arcgis/products/arcgis-cityengine/overview), [Bentley OpenRoads Designer](https://www.bentley.com/software/openroads-designer/) and [Autodesk Civil3D](https://www.autodesk.com/products/civil-3d/features) are competent engineering tools but have an extremely high economic barrier for entry and are far better suited for detailed design where CST is looking at concept development and representation. 

The tool also looks at [LaneMaker](https://guotata1996.github.io/lanemaker/) and [ASAM OpenDRIVE](https://www.asam.net/standards/detail/opendrive/) 

ESRI's Computer Generate Architecture allows for rule based scripting of urban roads, this repo aims to replicate some of that functionality for street design. Mainly the work done under [Complete-Street-Rule](https://github.com/d-wasserman/Complete_Street_Rule) also allows for poaching some of the textures and assets.
